# Reflection

## Decisions

One of the biggest decisions I made during this project was changing what the comparison tool was actually focused on. In the beginning, the website mostly compared which house had better numbers, like a lower price or more square footage. After testing it more, I realized that it was not very useful because buyers can already compare numbers on their own without needing a website.

The monthly cost inquiry helped to make sense of the project as a whole; previously, there was just a comparison of the listing price without consideration for Property Tax, HOA Fees, Interest Rate, and Down Payment. Now, the comparison can be made between homes over time, giving a more accurate representation of how much money one would spend each month on both types of homes based on what they paid for them.
Another decision I made was to keep the dark color palette and gold accents throughout the site. I wanted the website to feel more modern and connected to real estate instead of looking plain or overly simple.

## What Worked

The monthly cost feature worked better than I expected and became the strongest part of the project. It made the comparison tool feel more useful and gave the website an actual purpose besides just comparing numbers.

I also think the overall design turned out well. The pages feel consistent, the layout is organized, and the property images helped the homepage feel less empty. I liked that the final version looked much more polished than the earlier versions of the project.

The compare page also ended up becoming more interactive than I originally planned. Adding score cards, charts, comparison history, and the print feature made the project feel more complete without making it overly complicated.

## Things I would have changed

I wish I could have spent time implementing all aspects of JS correctly before altering the overall design of my project, rather than the design of my project after I altered the JS and made a mess of it. I made many different CSS changes while working through problems with JS functions/events and this prevented me from being able to identify issues with one or the other, due to the fact that I addressed "x" first before addressing "y".

Another problem I encountered during this project was that I could have planned the scope of this project much better than I did. While working on this project, I developed new ideas for new things to add to it or how I wanted it to function before I completed my original ideas from the beginning. This resulted in changing the overall direction of this project several times while creating the site.


## Things I learned

If I could do the whole project again, I would have worked harder to ensure that JavaScript is fully functional before I made any changes to the way the project looked. Making all the changes I made to the CSS while solving problems related to JavaScript events/functions has made it very difficult to solve one issue while another is still unresolved.

Another issue I discovered as I worked on this project was how I could have much better planned the scope of the project than I did. I had developed new ideas for how I wanted to expand or work on the project before I actually finished with what I wanted to add to the project from start to finish. Throughout the development process, there were situations where I was expending greater effort thinking about how to design and present visually a site than I was developing that site.

Another important point of learning for me with this project was how much value there is in small design elements of any web page, i.e., spacing, font, color, and picture. Individually, these elements may seem trivial, but when added together, they can greatly impact the professional look and feel of a site. Consideration of these design elements has produced a significantly more cohesive result than what I had when I first started developing this website.
